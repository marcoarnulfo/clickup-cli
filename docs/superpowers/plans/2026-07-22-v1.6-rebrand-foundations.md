# v1.6 "Rebrand & foundations" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Lay the technical foundations for the whole roadmap (service layer, unified HTTP core with rate limiting, clock injection, error-with-context, a filtered-tasks endpoint, a cobra-based headless `report`) and rebrand the binary `clickup` → `clup` safely.

**Architecture:** A cobra root (`internal/cli`) either launches the existing bubbletea TUI (no subcommand) or runs headless subcommands. The report I/O pipeline moves from `internal/tui` into `internal/service`, consumed by both the TUI and headless. `internal/report`/`internal/duration` stay pure. The `internal/clickup` client unifies `get`/`post` into one `do(method,…)` gated by a shared rate limiter with `Retry-After` handling.

**Tech Stack:** Go 1.26, bubbletea/lipgloss/bubbles, cobra (`github.com/spf13/cobra`), `golang.org/x/time/rate`, `gopkg.in/yaml.v3`, stdlib `net/http`, `httptest`.

## Global Constraints

- `internal/report` and `internal/duration` stay **pure**: no I/O, no config/clickup imports, no internal `time.Now` (callers pass `now`).
- ClickUp API v2: `Authorization: <token>` (NO `Bearer`); epochs/durations in **ms**; negative duration = running timer (discarded).
- Everything in-repo in **English** (identifiers, comments, UI strings, tests, commit messages). Conventional Commits. **NEVER** `Co-Authored-By`.
- TDD: table-driven; `httptest` for the client/service; TUI via `Update()` + simulated msgs. `gofmt -l .`, `go vet ./...`, `go build ./...`, `go test ./... -race` all clean before each commit.
- Demo mode (`CLICKUP_DEMO=1`) does **NO I/O**; the service is never used on the demo path.
- The Go module path `github.com/marcoarnulfo/clickup-cli` does **not** change.
- Do **not** add `put`/`delete` client wrappers in v1.6 (no callers → dead code). `do` takes a method string so they are a trivial later add.

```go
func (c *Client) do(ctx context.Context, method, path string, query url.Values, body, out any) error {
	for attempt := 0; ; attempt++ {
		if err := c.limiter.Wait(ctx); err != nil {
			return err
		}
		var reader io.Reader
		if body != nil {
			buf, err := json.Marshal(body)
			if err != nil { return err }
			reader = bytes.NewReader(buf)
		}
		req, err := http.NewRequestWithContext(ctx, method, c.BaseURL+path, reader)
		if err != nil { return err }
		req.Header.Set("Authorization", c.token)
		// Preserve current behavior: post() set Content-Type on every POST, even a
		// nil body (e.g. StopTimer). Keep it for any POST or any body-carrying request.
		if body != nil || method == http.MethodPost {
			req.Header.Set("Content-Type", "application/json")
		}
		if len(query) > 0 { req.URL.RawQuery = query.Encode() }
		resp, err := c.http.Do(req)
		if err != nil { return err }
		bodyBytes, _ := io.ReadAll(resp.Body)
		resp.Body.Close()
		if resp.StatusCode == http.StatusTooManyRequests {
			if attempt >= maxRetries {
				return fmt.Errorf("clickup API 429: rate limit exceeded after %d attempts", attempt)
			}
			wait := retryDelay
			if d, ok := retryAfter(resp.Header); ok { wait = d }
			if wait > retryAfterClamp { wait = retryAfterClamp } // clamp (var, overridable in tests)
			select {
			case <-ctx.Done(): return ctx.Err()
			case <-time.After(wait):
			}
			continue // retry re-consumes the limiter at loop top
		}
		return finishJSON(resp.StatusCode, bodyBytes, out)
	}
}

// get keeps the map ergonomics of the old API (skipping empty values).
func (c *Client) get(ctx context.Context, path string, query map[string]string, out any) error {
	var q url.Values
	if len(query) > 0 {
		q = url.Values{}
		for k, v := range query { if v != "" { q.Set(k, v) } }
	}
	return c.do(ctx, http.MethodGet, path, q, nil, out)
}
func (c *Client) post(ctx context.Context, path string, body any, out any) error {
	return c.do(ctx, http.MethodPost, path, nil, body, out)
}
```
